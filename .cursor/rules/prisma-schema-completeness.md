# Prisma Schema Completeness Rule

## CRITICAL: Always Verify Schema Changes in ALL Schema Files

### Understanding the Schema Architecture

This project uses a **multi-file schema organization**:

**Individual Schema Files (SOURCE - Edit These):**
- `prisma/schema/core.prisma` - Core system models (User, Role, Institution)
- `prisma/schema/academic.prisma` - Academic models (Course, Subject, Class)
- `prisma/schema/users.prisma` - User-related models (Student, Teacher, Parent)
- `prisma/schema/fees.prisma` - Fee management models
- `prisma/schema/communication.prisma` - Messaging and notification models
- `prisma/schema/library.prisma` - Library management models
- `prisma/schema/staff.prisma` - Staff management models
- `prisma/schema/gatepass.prisma` - Gatepass system models
- `prisma/schema/timetable.prisma` - Timetable and scheduling models
- `prisma/schema/analytics.prisma` - Analytics and reporting models
- `prisma/schema/assessment.prisma` - Assessment and grading models
- `prisma/schema/records.prisma` - Academic records models
- `prisma/schema/question-paper.prisma` - Question paper models

**Generated Schema File (AUTO-GENERATED - Never Edit):**
- `prisma/schema.prisma` - Combined file created by `scripts/build-schema.js`

### Problem
When modifying Prisma models, changes must be made in the **individual schema files** in `/prisma/schema/`, not the generated `schema.prisma` file.

### Build Process Flow

```bash
# 1. Edit individual files
vim prisma/schema/core.prisma

# 2. Build combined schema
npm run build:schema  # Runs scripts/build-schema.js

# 3. Use generated schema
npx prisma generate   # Uses prisma/schema.prisma
npx prisma migrate    # Uses prisma/schema.prisma
```

### Rule: Schema Change Checklist

**BEFORE making any Prisma model changes:**

1. **Identify the correct schema file:**
   - User, Role, Institution models → `prisma/schema/core.prisma`
   - Course, Subject, Class models → `prisma/schema/academic.prisma`
   - Student, Teacher, Parent models → `prisma/schema/users.prisma`
   - Fee-related models → `prisma/schema/fees.prisma`
   - Communication models → `prisma/schema/communication.prisma`
   - Library models → `prisma/schema/library.prisma`
   - Staff models → `prisma/schema/staff.prisma`
   - Gatepass models → `prisma/schema/gatepass.prisma`
   - Timetable models → `prisma/schema/timetable.prisma`
   - Analytics models → `prisma/schema/analytics.prisma`
   - Assessment models → `prisma/schema/assessment.prisma`
   - Records models → `prisma/schema/records.prisma`
   - Question paper models → `prisma/schema/question-paper.prisma`

2. **Make changes in the individual file:**
   - ❌ NEVER edit `prisma/schema.prisma` directly (it's auto-generated by build-schema.js)
   - ✅ ALWAYS edit the specific file in `prisma/schema/`

3. **Build the combined schema:**
   ```bash
   npm run build:schema
   # This runs scripts/build-schema.js to combine all individual files
   ```

4. **Verify the change appears in generated schema:**
   ```bash
   # Check that prisma/schema.prisma contains your changes
   grep -n "your_new_field" prisma/schema.prisma
   ```

5. **Test Prisma client generation:**
   ```bash
   npx prisma generate
   # Should succeed without TypeScript errors
   ```

### Common Mistakes to Avoid

❌ **Adding fields to generated `schema.prisma`** - changes will be overwritten by build-schema.js
❌ **Forgetting to run `npm run build:schema`** after modifying individual files
❌ **Not checking if the field appears in the final combined schema**
❌ **Assuming deployment issues are always script-related** - check schema completeness first
❌ **Editing the wrong domain file** - User fields go in core.prisma, not users.prisma
❌ **Not understanding the build process** - Individual files → build-schema.js → combined schema

### Verification Commands

```bash
# 1. Build schema from individual files
npm run build:schema

# 2. Check if your field exists in generated schema
grep -n "institutionId" prisma/schema.prisma

# 3. Generate Prisma client
npx prisma generate

# 4. Check TypeScript compilation
npm run build
```

### When Schema Changes Fail

**FIRST check:** Is the field actually in the generated `prisma/schema.prisma`?
- If NO → You either:
  - Edited `prisma/schema.prisma` directly (wrong - it gets overwritten)
  - Edited the wrong individual file in `prisma/schema/`
  - Forgot to run `npm run build:schema` after editing
  - Made a syntax error in the individual file
- If YES → Then check deployment scripts, Docker build, etc.

**Debug steps:**
1. Check which individual file you edited
2. Verify the syntax is correct in that file
3. Run `npm run build:schema` and check for errors
4. Confirm the field appears in the generated `prisma/schema.prisma`
5. Only then troubleshoot deployment/build issues
